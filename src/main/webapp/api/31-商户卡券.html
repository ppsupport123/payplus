3.1 商户卡券

3.用户账户营销

此版块重点介绍用户账户的营销应用功能，下图将详细介绍商家调用流程（包含卡券和红包）：

3.1商户卡券

3.1.1创建卡券

【场景介绍】

 商户可创建一次营销活动的卡券模板，暂时只支持固定金额的抵扣模式，即直接抵扣支付订单金额。调用该接口时商户需传卡券类型为自有资金型或信息流型的参数字段。暂时只支持两级商户模式即平台模式，发券方可为平台或分账方，其中分账方目前只能发信息流型的卡券。

备注：其中创建自有资金型卡券时，需要冻结对应卡券的资金。

请求地址：

环境 接口服务URI
生产环境 \/rest\/v1.0\/merchant\/createCouponDefine

3.1.1.1请求参数：

参数名称 参数含义 数据类型 必填 参数说明
requestNo 商户请求号 String(32) Y
sendMerchantNo 创券方商户编号 String(16) Y
name 活动名称 String(32) Y
value 面值 Amount Y 固定金额,单位：元限制大于等于0.01，最大18位
fundType 资金类型 Enum (4) Y 信息型-NONE
totalCount 数量 Int Y 限定大于等于1的正整数
isAutoRecevie 是否自动领取 Boolean Y 是-true否-false

3.1.1.2返回参数：

参数名称 参数定义 参数说明
code 返回码 请参考附录：返回码列表
message 返回消息 返回码的详细说明

3.1.2发送卡券

【场景介绍】

将已创建的卡券发送给商户自己的用户，建立起卡券与用户的对应关系。

请求地址：

环境 接口服务URI
生产环境 \/rest\/v1.0\/merchant\/sendCoupon

3.1.2.1请求参数

参数名称 参数含义 数据类型 必填 参数说明
defineRequestNo 卡券定义编号 String(32) Y
merchantNo 商户编号 String(16) Y 用户所属商户的商编
merchantUserId 领券方商户用户标识 String(32) Y
expireTime 卡券领取过期时间 Int (1,44640) Y 卡券发送后未领取自动撤回的过期时间，单位（分钟）；限定1<=Time<=44640(31天)

3.1.2.2返回参数

参数名称 参数定义 参数说明
couponNo 卡券编号
code 返回码 请参考附录：返回码列表
message 返回消息 返回码的详细说明

3.1.3领取卡券

【场景介绍】

卡券的使用需要用户先确认领取时，需要调用此接口

请求地址：

环境 接口服务URI
生产环境 \/rest\/v1.0\/user\/receiveCoupon

3.1.3.1请求参数:

参数名称 参数含义 数据类型 必填 参数说明
couponNo 卡券编号 String(32) Y
merchantNo 商户编号 String(16) Y 用户所属商户的商编
merchantUserId 领券方商户用户标识 String(32) Y

3.1.3.2返回参数:

参数名称 参数定义 参数说明
code 返回码 请参考附录：返回码列表
message 返回消息 返回码的详细说明

3.1.4用户未领取卡券查询

【场景介绍】

查询某个用户未领取的所有卡券列表。

备注：如传入创建的卡券定义编号，则查询的是该卡券的未领取信息。

请求地址：

环境 接口服务URI
生产环境 \/rest\/v1.0\/user\/queryUnReceivedCoupons

3.1.4.1请求参数:

参数名称 参数含义 数据类型 必填 参数说明
defineRequestNo 卡券定义编号 String(32) N 非必填参数，如果不传则查询当前用户名下所有未领取卡券信息
merchantNo 商户编号 String(16) Y 用户所属商户的商编
merchantUserId 领券方商户用户标识 String(32) Y

3.1.4.2返回参数:

参数名称 参数定义 参数说明
couponList 卡券列表 卡券列表说明详见下文注释
code 返回码 请参考附录：返回码列表
message 返回消息 返回码的详细说明

卡券列表注释说明：

1、格式：Json串。

2、返回示例：

[{

"couponNo" : "AMCRP20160830111448823NpdpGrj",

"couponType" : "COUPON",

"fundType" : "NONE",

"sendMerchantNo" : "BL12345678901260",

"name" : "100000",

"value" : "0.01",

"status" : "RECEIVED",

"expireTime" : "2016-08-30 11:19:48"

}]

即

[{

"卡券编号" : "AMCRP20160830111448823NpdpGrj",

"卡券类型" : "COUPON",

"资金类型" : "NONE",

"发券方商户编号" : "BL12345678901260",

"活动名称" : "100000",

"面值" : "0.01",

"状态" : "RECEIVED",

"未领取卡券过期时间" : "2016-08-30 11:19:48"

}]

3、卡券状态：初始化-INIT、处理中-PROCESSING、发放中-SENDING、已退回-REFUND、已领取-RECEVIED。

4、卡券类型：卡券-COUPON。

【场景介绍】

请求地址：

环境 接口服务URI
生产环境 \/rest\/v1.0\/user\/queryAllCoupons

3.1.5.1请求参数:

参数名称 参数含义 数据类型 必填 参数说明
merchantNo 商户编号 String(16) Y 用户所属商户的商编
merchantUserId 领券方商户用户标识 String(32) Y

3.1.5.2返回参数:

参数名称 参数定义 参数说明
couponList 卡券列表 卡券列表说明详见下文注释
code 返回码 请参考附录：返回码列表
message 返回消息 返回码的详细说明

卡券列表注释说明：参考3.1.4的注释说明

3.1.6用户可用卡券查询

【场景介绍】

用户在确认支付前，查询已领取的全部可用卡券。

备注：返回可用卡券列表时，按照订单金额维度，分为已领取可用和已领取不可用两个列表。

请求地址：

环境 接口服务URI
生产环境 \/rest\/v1.0\/user\/queryAvailableCoupons

3.1.6.1请求参数

参数名称 参数含义 数据类型 必填 参数说明
merchantNo 商户编号 String(16) Y 用户所属商户的商编
merchantUserId 领券方商户用户标识 String(32) Y
orderAmount 订单金额 Amount Y 必须大于等于0.01，最大18位

3.1.6.2返回参数

参数名称 参数定义 参数说明
code 返回码 请参考附录：返回码列表
message 返回消息 返回码的详细说明
availableCouponList 已领的可用卡券列表 不包含未领、过期等不可用的，即卡券面值小于等于订单金额的有效卡券列表。
unAvailableCouponList 已领的不可用卡券列表 不包含未领、过期等不可用的，即卡券面值大于订单金额的有效卡券列表。

卡券列表注释说明：参考3.1.4的注释说明

3.1.7卡券定义信息查询

【场景介绍】

查询已创建卡券目前的整体情况，包括总金额、数量、面值、活动名称等。

请求地址：

环境 接口服务URI
生产环境 \/rest\/v1.0\/merchant\/queryCouponDefine

3.1.7.1请求参数:

参数名称 参数含义 数据类型 必填 参数说明
defineRequestNo 卡券定义编号 String(32) Y
sendMerchantNo 创券方商户编号 String(16) Y

3.1.7.2返回参数:

参数名称 参数定义 参数说明
couponType 卡券类型 抵扣卡券：COUPON
fundType 资金类型 信息型：NONE
name 活动名称
value 面值 单位：元
totalAmount 总金额
totalCount 总数量
unSendCount 未发送数量
unreceivedCount 未领取数量
status 活动状态 初始化-INIT
refundAmount 退款金额 单位：元，目前返回为空
freezeAmount 冻结资金金额 单位：元，目前返回为空
code 返回码 请参考附录：返回码列表
message 返回消息 返回码的详细说明
