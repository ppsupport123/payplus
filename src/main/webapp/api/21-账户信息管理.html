2.1 账户信息管理

2.1.1用户账户注册（会员注册）

【场景介绍】

当有用户资金账户的场景时，比如用于充、转、提功能，需要调用本接口创建用户，商户用户标识是用户在商户平台的唯一标识，一个用户标识+身份证号只允许注册一次。

【调用流程】

【请求地址】

环境 接口服务URI
生产环境 /rest/v1.0/user/register

2.1.1.1请求参数

参数名称 参数含义 数据类型 必填 参数说明
requestNo 商户请求号 String(32) Y
merchantNo 商户编号 String(16) Y
merchantUserId 商户用户标识 String(32) Y

2.1.1.2返回参数

参数名称 参数定义 参数说明
memberNo 会员编号 商户用户在钱麦的会员编号
code 返回码 请参考附录：返回码列表
message 返回消息 返回码的详细说明

2.1.2发起实名认证

【场景介绍】

根据相关法规要求，用户资金账户的充值、提现、转账、账户余额支付、绑卡支付时，需要实名认证后才可以使用，实名认证时根据传入不同的终端类型跳转到钱麦不同的页面，用户按照页面提示自行操作完成。

【重要说明】目前用户的实名认证是通过对其银行卡信息的认证完成的，用户实名认证通过之后，会引导用户设置支付密码，以保证支付账户的安全性，该操作和实名操作都是在钱麦的H5页面完成，设置完成之后，结果同步\/异步反馈给商户，且在用户操作完成之后，返回商户端页面。

2.1.2.3主动通知：

  1. 通知方式：将参数通知到请求时传入的serverCallbackUrl。

  2. 通知机制：仅异步通知一次，如通知失败，则每3分钟补偿通知1次，共通知3次。

  3. 通知参数：同4.1.2.2订单查询的返回参数。

2.1.3用户证件上传

【场景介绍】

  1. 根据相关法规要求，第三方支付公司对用户身份信息验证的内容及渠道，与账户的累计支付限额有相关性，故用户上传较多的身份信息，第三方支付公司便可通过较多渠道对用户身份进行认证，其账户的等级和权限就会较高。

  2. 第三方支付公司留存用户的身份证影印件也是法规要求，故尽量引导用户上传。

【重要说明】 用户的该操作需要在钱麦页面完成，故需要商户端向钱麦发起用户资质上传的请求，钱麦将结果同步\/异步反馈给商户，且在用户操作完成之后，返回商户端页面。
